關於圖片存讀功能

這個功能的起因是由於一個製造業的客戶需求，其開報工檢驗的時候，需要紀錄不良品的圖片，直接用APP上傳到SERVER。 後續也需要用網頁追蹤，所以使用FileStream的功能，搭配寫好的API，將圖片檔案做儲存、讀取、刪除的管理，其詳細代碼可參考我的GitHun(已去除可識別資料)。

FileStream設置

首先資料庫必須要啟用FileStream功能。然後於SERVER建立IIS服務站台，最後將API發佈到SERVER上。

實際呼叫的範例

API 與測試方式

檔案上傳 /api/fs/upload

檔案下載 /api/fs/download

檔案刪除 /api/fs/delete

API 與測試方式

DB: FileStore @ localhost sa / XXXXX

Server URL: http://localhost/api/fs

KEY: 請在 Header 中加入 API_KEY: @@@API-KEY@@@

檔案上傳 /api/fs/upload

Action: POST

Body: Mutipart

File 檔案路徑

ContentType 檔案的MIME-TYPE

RefNo 給應用程式利用的參考

Return: File ID (fs_id) 檔案存取索引ID
